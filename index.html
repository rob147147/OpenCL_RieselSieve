Project Overview

This project is an attempt to port the popular srsieve program to OpenCL in order to utilise the power of GPUs. The current version of this code uses Java for the CPU-side and JOCL to compile and execute the kernel on a GPU.

Purpose

The purpose of this sieve program (and srsieve) is to find small(ish) factors of numbers of the form k*b^n-1. Finding a factor for a particular expression saves performing a more expensive LLR test in order to see if the expression is prime.

Algorithm

The main chunk of the kernel involves solving a discrete logarithm problem, and the algorithm used here is Pollard's Kangaroo(sometimes called Lambda) algorithm. This is a probabilistic algorithm, and thus can miss occasional factors if it is not executed for enough loops. Once the discrete log has been solved the remainder of the algorithm checks for any matches with the candidates we are interested in.

All multiplications are done in montgomery space to remove computationally expensive modulo operations, and recent kernels use mul24/mad24 operations which should be considerably quicker on recent AMD GPUs.

Performance

Performance is currently reasonable on small test cases on my ATI HD4850, but on larger test cases performance does not yet match that of a recent CPU. A small number of tests have been carried out on GCN based AMD GPUs but performance is currently worse than the HD4850. This seems to be caused by high VGPR usage reducing the number of waves in flight per SIMD.

Without regular access to a GCN card it is difficult to debug why performance is so poor. If anyone with a GCN card and CodeXL would like to help provide analysis and feedback please feel free to get in touch.
